# 拾光 v0.9.1 发布后交付引导升级

## 目标

解决用户发布云端礼物后不知道去哪里找链接、也不知道应该把哪一种内容发送给收件人或朋友的问题。

## 新流程

1. 用户点击“确认并正式发布”。
2. 发布成功后自动打开“交付中心”。
3. 交付中心明确区分：
   - 正式礼物链接：发给收件人。
   - 秘密共创链接：发给朋友或家人。
4. 每种链接都提供可直接复制的话术和检查入口。
5. 用户关闭弹窗后，制作台继续显示“这份礼物已经发布”的下一步提醒。

## 更新源码

本版本建立在 v0.9 上，不需要新增数据库迁移。

用本项目覆盖现有 v0.9 源码后执行：

```powershell
npm install
npm run check
git add -A
git commit -m "Upgrade to v0.9.1 delivery guidance"
git push origin main
```

## 验收

- 未发布时，“专属链接 / 交付”按钮不可用。
- 正式发布成功后，交付中心自动打开。
- “发给收件人”区域显示正式礼物链接、二维码、测试按钮和发送话术。
- “发给朋友 / 家人”区域可以生成秘密共创链接和群聊邀请话术。
- 关闭交付中心后，制作台仍显示醒目的交付提醒。
- 再次打开已经发布的礼物时，可以从顶部“专属链接 / 交付”或提醒卡重新进入。

## 注意

- 正式礼物链接不要发给共创参与者。
- 共创邀请链接不要直接发给收件人，以免提前泄露惊喜。
- 再生成一个共创链接不会自动撤销旧链接；旧链接应在“多人共创”步骤中手动撤销。
