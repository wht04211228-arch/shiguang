# 拾光 v0.9.2 升级说明

本版本解决两个问题：

1. 制作需求问卷提交后，内容自动进入对应订单的制作台；
2. 制作台原有的大面积顶部进度横条改为轻量状态栏，不再遮挡编辑视野。

## 问卷自动导入内容

用户正式提交制作需求后，会直接进入：

```text
/studio?order=订单ID&brief=imported
```

系统自动带入：

- 收件人称呼
- 使用场景
- 视觉主题
- 真实回忆
- 信件重点
- 基于关系与语气生成的开场文案

以下信息仅显示在制作台的私有需求卡中，不会出现在收件人礼物里：

- 双方关系
- 文案语气
- 期望交付日期
- 联系方式
- 禁忌内容
- 其他特殊要求

如果用户已经在制作台编辑过内容，再次提交或修改问卷时，会同步问卷字段，同时保留已上传照片和非问卷回忆。

## 轻量进度导航

原先固定占据整行的制作完成度和情感丰富度区域已删除，改为编辑区顶部的紧凑状态栏：

- 圆环显示制作完成度
- 显示当前步骤
- 情感丰富度使用小标签展示
- 点击“查看进度”才展开完整建议
- 手机端不再占用大面积纵向空间

## 升级方式

本版本不需要运行新的 Supabase 数据库迁移。

将 v0.9.2 源文件覆盖到当前项目后运行：

```powershell
npm install
npm run check
git add -A
git commit -m "Upgrade to v0.9.2 brief sync and compact progress"
git push origin main
```

## 验收步骤

1. 使用已付款订单打开制作需求问卷；
2. 填写至少两条真实回忆并正式提交；
3. 页面应自动进入对应订单的制作台；
4. 检查收件人、场景、主题、回忆和信件是否已带入；
5. 检查禁忌内容只出现在私有需求卡中；
6. 检查制作台顶部不再显示大面积横向进度条；
7. 点击“查看进度”，确认完整进度与下一步建议正常展开。
