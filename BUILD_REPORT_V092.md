# 拾光 v0.9.2 构建报告

## 版本目标

- 制作需求问卷自动导入制作台
- 避免客户重复输入相同内容
- 将固定大面积进度横条替换为紧凑进度导航
- 保留原有人工付款、共创、发布交付和收件人体验功能

## 自动导入规则

| 问卷字段 | 制作台位置 |
|---|---|
| 收件人 | 礼物基础信息 |
| 使用场景 | 礼物基础信息 |
| 视觉方向 | 视觉主题 |
| 真实回忆 | 回忆故事 |
| 一定要出现的内容 | 专属信件 |
| 关系和语气 | 用于生成开场文案，并保留在私有需求卡 |
| 禁忌内容 | 仅私有需求卡可见 |
| 特殊要求 | 仅私有需求卡可见 |

## 安全设计

- 禁忌内容、联系方式和制作备注不会写入公开贺卡数据；
- 已有编辑内容再次同步时，保留用户上传的照片和非问卷回忆；
- 本地草稿按订单隔离，避免不同客户的内容互相覆盖；
- 已存在云端礼物时优先载入云端礼物，再允许用户手动同步最新问卷。
