# 拾光 v0.9.4 动态制作旅程升级说明

## 本版本解决的问题

旧制作台固定展示十个步骤，不同套餐、不同权益和不同阶段看到的流程完全相同。v0.9.4 改为根据真实状态动态生成下一步，避免用户面对与自己无关的功能。

## 新增能力

### 动态制作阶段

系统综合以下信息生成制作旅程：

- 当前套餐；
- 多人共创人数权益；
- 视频权益；
- 已填写内容；
- 共创投稿状态；
- 是否完成收件人视角模拟；
- 礼物是否已经发布；
- 是否已经收到回复。

轻定制不会把未购买的共创和视频功能放进主流程；深度定制与私人策划会根据实际权益显示对应任务。

### 主推荐与并行任务

制作台始终显示：

- 1 个当前最重要的主推荐；
- 最多 2 个可以同时完成的备选任务；
- 推荐原因；
- 预计耗时；
- 完成后的结果。

等待朋友投稿时，系统会提示用户先完善信件、媒体或开放规则，不会让流程停住。

### 发布后自动切换阶段

礼物发布后，制作旅程自动切换为：

1. 检查正式礼物；
2. 打开交付中心并发送链接；
3. 继续收集共创内容（如已购买）；
4. 查看收件人回复；
5. 查看体验数据。

### 查看完整旅程

“完整旅程”会展示当前套餐下实际存在的任务、完成状态、等待状态和可选增强项。

### 查看全部功能

动态旅程只负责推荐，不会删除已有功能。用户可以随时打开“全部功能”，进入视觉主题、回忆、信件、共创、媒体、未来约定、互动惊喜、开放时间和发布检查。

### 收件人视角模拟记录

从动态旅程启动模拟后，完成模拟会记录在当前订单的本机流程状态中，系统随后推荐进入发布检查。

## 更新方式

将补丁文件覆盖到当前 v0.9.3 项目根目录，然后执行：

```powershell
npm install
npm run check
git add -A
git commit -m "Upgrade to v0.9.4 dynamic studio journey"
git push origin main
```

## 数据库与环境变量

本次升级：

- 不需要运行 Supabase 数据库迁移；
- 不需要新增 Vercel 环境变量；
- 不会删除现有订单、贺卡、付款凭证和共创投稿。
